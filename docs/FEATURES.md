# 기능 설명

## 1. Home 화면

Home 화면은 서비스의 메인 화면입니다. 등록된 도서 중 좋아요 수가 높은 도서를 인기 도서로 보여줍니다.

### 기능

- 좋아요 수 기준 인기 도서 표시
- 전체 도서 목록 페이지 이동
- 등록된 도서가 없을 경우 빈 상태 안내

---

## 2. 도서 목록 화면

도서 목록 화면은 `GET /books`로 불러온 데이터를 카드 형태로 렌더링합니다.

### 기능

- 도서 카드 목록 표시
- 도서 제목/작가 기준 검색
- 검색 결과 없음 안내
- 등록된 도서 없음 안내
- 카드 클릭 시 상세보기 모달 표시
- 상세보기 모달에서 좋아요/조회수/수정/삭제 기능 제공

---

## 3. 도서 상세보기 모달

도서 카드를 클릭하면 상세보기 모달을 표시합니다.

### 표시 정보

- 도서 제목
- 작가
- 도서 내용
- 표지 이미지
- 좋아요 수
- 조회수
- 생성일/수정일

### 제공 기능

- 상세보기 시 조회수 증가
- 좋아요 증가
- 수정 화면 이동
- 삭제 확인 후 삭제

---

## 4. 도서 등록 화면

도서 등록 화면은 신규 도서 정보를 입력하고 저장하는 기능을 제공합니다.

### 입력 항목

- 도서 제목
- 작가 이름
- 도서 내용
- OpenAI API Key
- 이미지 품질 옵션
- 이미지 크기 옵션

### 기능

- 제목/작가/내용 입력
- 필수 입력값 검증
- AI 표지 이미지 생성
- 이미지 생성 비용 안내
- 생성 이미지 미리보기
- 등록하기 버튼으로 신규 도서 저장
- 등록 시 좋아요/조회수 초기값 설정

### 이미지 생성과 등록 동작 분리

```text
이미지 생성하기 클릭
→ OpenAI Images API 호출
→ 생성된 이미지를 미리보기로 표시

등록하기 클릭
→ 현재 입력값과 현재 coverImageUrl을 기준으로 POST /books 수행
```

이미지 생성에 실패해도 도서가 자동 등록되지 않도록 이미지 생성과 등록 동작을 분리했습니다.

---

## 5. 도서 수정 화면

도서 수정 화면은 `/books/:id`로 기존 도서 데이터를 조회한 뒤 수정 폼에 반영합니다.

### 기능

- 기존 도서 데이터 조회
- 기존 표지 이미지 표시
- 제목/작가/내용 수정
- 이미지 품질 선택
- 이미지 재생성
- 수정 완료 시 PATCH 요청
- 수정 후 목록 화면 이동

---

## 6. AI 표지 이미지 생성 기능

AI 표지 이미지는 도서 제목, 작가, 내용을 기반으로 생성합니다.

### 현재 실행 구조

```text
React
→ OpenAI Images API 직접 호출
→ b64_json 응답
→ React에서 data:image/png;base64,... 변환
→ coverImageUrl 상태 저장
→ 등록/수정 시 db.json에 저장
```

### 구현 특징

- API Key는 화면 입력값으로 받습니다.
- API Key는 코드에 하드코딩하지 않습니다.
- 이미지 생성 중에는 로딩 이미지를 보여줍니다.
- 이미지 생성 실패 시 에러 이미지를 미리보기로 보여줍니다.
- 등록/수정 저장 시 로딩/에러 이미지는 실제 표지로 저장하지 않도록 기본 이미지로 대체합니다.

---

## 7. 삭제 기능

상세보기 모달에서 삭제 버튼을 누르면 확인 창을 표시한 뒤 도서를 삭제합니다.

```http
DELETE /books/:id
```

삭제 후 목록 상태에서도 해당 도서를 제거합니다.

---

## 8. 좋아요 기능

상세보기 모달에서 좋아요 버튼을 누르면 해당 도서의 좋아요 수가 증가합니다.

```http
PATCH /books/:id
```

요청 본문에는 증가된 `likes` 값만 담습니다.

---

## 9. 조회수 기능

도서 카드를 클릭하여 상세보기 모달을 열면 조회수가 증가합니다.

```http
PATCH /books/:id
```

요청 본문에는 증가된 `views` 값만 담습니다.

---

## 10. 검색 기능

도서 목록 화면에서 검색어를 입력하면 도서 제목 또는 작가명을 기준으로 목록을 필터링합니다.

### 처리 상황

| 상황 | 안내 |
|---|---|
| 등록된 도서 없음 | 등록된 도서가 없습니다. |
| 검색 결과 없음 | 검색 결과가 없습니다. |

---

## 11. 이미지 저장 처리

| 상황 | 저장 값 |
|---|---|
| AI 이미지 생성 성공 | `data:image/png;base64,...` |
| 이미지가 비어 있음 | `/noImage.jpg` |
| 로딩 이미지 상태 | `/noImage.jpg` |
| 에러 이미지 상태 | `/noImage.jpg` |

---

## 12. 주요 주의사항

### OpenAI API Key

현재 프로젝트는 프론트엔드 단독 학습 구조이므로 사용자가 화면에 입력한 API Key로 OpenAI API를 직접 호출합니다. API Key를 코드, `.env`, README, GitHub에 올리면 안 됩니다.

### Payload 용량

base64 Data URL은 `db.json`에 저장할 수 있지만, 이미지 크기나 품질이 커지면 요청 용량이 커져 저장이 실패할 수 있습니다. 이 경우 이미지 크기 또는 품질을 낮추는 방식으로 대응합니다.
