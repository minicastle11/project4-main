# 기능 설명

## 1. Home 화면

Home 화면은 서비스의 메인 화면입니다. 좋아요 수를 기준으로 인기 도서 목록을 보여주고, 전체 도서 목록 페이지로 이동할 수 있습니다.

### 기능

- 좋아요 수 기준 도서 정렬
- 인기 도서 목록 표시
- 전체 목록 페이지 이동
- 도서가 없을 경우 빈 상태 안내 문구 표시

---

## 2. 도서 목록 화면

도서 목록 화면은 `/books` 데이터를 기반으로 카드 목록을 렌더링합니다.

### 기능

- 도서 카드 목록 표시
- 도서 제목/작가 기준 검색
- 검색 결과 없음 안내
- 등록된 도서 없음 안내
- 카드 클릭 시 상세보기 모달 표시
- 상세보기 모달에서 좋아요/조회수/수정/삭제 기능 제공

---

## 3. 도서 상세보기 모달

도서 카드를 클릭하면 상세보기 모달을 표시합니다.

### 표시 정보

- 도서 제목
- 작가
- 도서 내용
- 표지 이미지
- 좋아요 수
- 조회수
- 생성일/수정일

### 제공 기능

- 좋아요 증가
- 조회수 증가
- 수정 화면 이동
- 삭제 확인 후 삭제

상세보기 모달 UI는 최종 제출을 위해 개선될 수 있습니다.

---

## 4. 도서 등록 화면

도서 등록 화면은 신규 도서 정보를 입력하고 저장하는 기능을 제공합니다.

### 입력 항목

- 도서 제목
- 작가 이름
- 도서 내용
- 표지 이미지
- 이미지 품질 옵션
- 이미지 크기 옵션

### 기능

- 제목/작가/내용 입력
- 필수 입력값 검증
- AI 표지 이미지 생성
- 이미지 생성 비용 안내
- 생성 이미지 미리보기
- 등록하기 버튼으로 신규 도서 저장
- 등록 시 좋아요/조회수 초기값 설정

### 이미지 생성과 등록 동작 분리

```text
이미지 생성하기 클릭
→ 이미지 생성만 수행
→ 생성된 이미지를 미리보기로 표시

등록하기 클릭
→ 현재 입력값과 현재 coverImageUrl을 기준으로 POST /books 수행
```

이미지 생성에 실패해도 도서가 자동 등록되지 않도록 분리되어 있습니다.

---

## 5. 도서 수정 화면

도서 수정 화면은 `/books/:id`로 기존 도서 데이터를 조회한 뒤 수정 폼에 반영합니다.

### 기능

- 기존 도서 데이터 조회
- 기존 표지 이미지 표시
- 제목/작가/내용 수정
- 이미지 품질 선택
- 이미지 크기 옵션
- 이미지 재생성
- 수정 완료 시 PATCH 요청
- `updatedAt` 갱신

---

## 6. AI 표지 이미지 생성 기능

AI 표지 이미지는 도서 제목과 내용을 기반으로 생성합니다.

### 현재 실행 구조

```text
React
→ api_mid_server.js
→ OpenAI Images API
→ b64_json 응답
→ React에서 data:image/png;base64,... 변환
→ coverImageUrl에 저장
→ 등록/수정 시 db.json에 저장
```

미들웨어 서버는 OpenAI API Key가 프론트엔드에 직접 노출되는 문제를 줄이기 위해 사용합니다.

---

## 7. 삭제 기능

상세보기 모달에서 삭제 버튼을 누르면 확인 창을 표시한 뒤 도서를 삭제합니다.

```http
DELETE /books/:id
```

삭제 후 목록 상태에서도 해당 도서를 제거합니다.

---

## 8. 좋아요 기능

상세보기 모달에서 좋아요 버튼을 누르면 해당 도서의 좋아요 수가 증가합니다.

```http
PATCH /books/:id
```

---

## 9. 조회수 기능

도서 카드를 클릭하여 상세보기 모달을 열면 조회수가 증가합니다.

```http
PATCH /books/:id
```

---

## 10. 빈 상태 처리

| 상황 | 안내 문구 |
|---|---|
| 등록된 도서 없음 | 등록된 도서가 없습니다. |
| 검색 결과 없음 | 검색 결과가 없습니다. |
| Home 인기 도서 없음 | 등록된 도서가 없습니다. |

---

## 11. 주요 주의사항

### OpenAI API Key 정책/보안 이슈

OpenAI API Key를 프론트엔드 코드나 브라우저 요청에 직접 노출하면 안 됩니다. 이를 우회하기 위해 `api_mid_server.js`와 `.env`를 사용합니다.

### Payload 용량 이슈

base64 Data URL은 `db.json`에 저장할 수 있지만, 이미지 크기나 품질이 커지면 Payload가 10000KB 이상으로 커져 저장이 실패할 수 있습니다.
